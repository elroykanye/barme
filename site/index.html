The object store that stores by content, not filename.

barme speaks the S3 API, then goes further. Every object is split into content-addressed chunks, so you get deduplication, cheap versioning, and integrity-checked reads without extra machinery. Self-hostable, single binary, written in Rust.

Storage properties, for free

Addressing objects by their content hash gives you the things you usually bolt on later.

Deduplication

Identical chunks share a hash and are stored once, across every object and every version. No duplicate bytes hit disk.

Cheap versioning

A new version only writes the chunks that changed. A 25 MB re-upload with an edited header can cost about 5 MB on disk. Old versions stay intact and addressable.

Integrity by default

Reads are verified against their hash. A corrupted byte is caught on access instead of surfacing later as silent data loss.

Efficient sync

Two nodes reconcile by comparing merkle tree roots and transferring only the branches that differ. Prove a chunk belongs to an object without shipping the whole thing.

Per-bucket compression

From byte-exact zstd to JPEG XL lossless transcode (JPEGs 20-30% smaller, rebuildable byte for byte) to visually-lossless AVIF. Set per bucket, recorded per object.

Semantic search

Query objects by meaning, text or image, over a vector index keyed by content hash. Built asynchronously off the write path, per-tenant isolated, and rebuildable from the stored bytes.

How a write works

An object is a merkle tree of content hashes, roughly how git handles blobs.

1

Chunk

The file is split into content-defined chunks with FastCDC, so a local edit only disturbs the chunks it touches.

2

Hash & dedup

Every chunk is addressed by its hash. Chunks already in the store are skipped; only new bytes are written.

3

Manifest

A manifest records the chunk list, codec, fidelity, and quality. Its hash is the object id, the merkle root.

4

Move pointer

A mutable bucket/key label points at the manifest. Rolling back is just pointing it at an older one.

Two doors, one engine

An object written over S3 can be inspected, diffed, and searched through the native API, because both call the same engine.

S3 API · port 9000

Object PUT, GET, DELETE, HEAD, listing, multipart, and presigned URLs with AWS SigV4. Point any S3 client or SDK at it with path-style addressing.

Native API · port 7373

The operations S3 has no vocabulary for: version diffs, fetch-by-hash, tree-based sync, fidelity introspection, and semantic search.

Compression tiers

Nothing is ever stored uncompressed. Each object records how it was stored, so defaults can change without orphaning old data.

TierCodecFidelityResult
